# 每日简报

从指定目录中读取某个日期的所有日志文件，汇总内容后向用户交付一份整合简报。

## 触发条件

用户请求某个目录中特定日期的每日简报 / 每日摘要 / daily brief。

## 执行步骤

1. **确定源目录** -- 如果用户未指定，使用已知的默认日志目录（如 `~/.hermes/human-like/thoughts/`）。

2. **列出匹配日期的文件** -- 搜索匹配以下模式的文件：
   - `YYYY-MM-DD.md`
   - `YYYY-MM-DD-*.md`
   使用 `search_files(target='files')` 或 `terminal` 枚举目标日期的文件。

3. **没有匹配文件** -- 明确告知用户：在 {目录} 中未找到 YYYY-MM-DD 的日志文件。

4. **读取所有匹配文件** -- 对每个文件使用 `read_file(path=..., limit=9999)`。注意优先级：
   - `YYYY-MM-DD.md` 是主日志，优先级最高
   - `YYYY-MM-DD-{SLUG}.md` 是子条目（例如 autonomous-exploration、micro-trigger、dream-thinking）

5. **在最终回复中汇总** -- 将内容整合成一份连贯的简报：
   - 如果文件有明确主题，按主题/来源分组
   - 如果文件按时间顺序或交错，按逻辑顺序呈现
   - 高亮关键决策、发现或待办事项
   - 保持简洁 -- 用户要的是信号，不是回放

## 规则

- **绝对不能修改源文件。** 只读操作。
- **绝对不能创建新文件。** 不生成摘要文件或写入磁盘 -- 直接在最终回复中交付。
- 如果文件很大，聚焦关键部分，不要 dump 原始内容。
- 如果多个文件覆盖重叠主题，在汇总中去重。
- 始终注明有哪些源文件参与了简报（列出文件名，方便用户追溯）。

## 常见陷阱

- 某些日期可能没有日志文件 -- 优雅处理，不要报错崩溃。
- 日期格式是 ISO 8601（`2026-04-05`），不是本地化格式。
- 带后缀的文件应标注来源（例如 "来自 autonomous-exploration: ..."）。
- 如果同一天有多个 cron 任务生成的文件，用户通常想要合并视图，而不是逐文件罗列。
